 Source 2 Viewer is a GUI application for Windows, there is also a 
command-line utility available for all Windows, Linux, and macOS.`n`nThe binary name is `nSource2Viewer-CLI`n.`n`nCommand-line options `n​`n`n`n`n`n`nOption`n`nDescription`n`n`n`n`n`n`nInput`n`n`n`n`n`n`n`n--input`n (or `n-i`n)`n`nInput file to be processed. With no additional arguments, a summary of the input(s) will be displayed.`n`n`n`n`n--recursive`n`n`nIf specified and given input is a folder, all sub directories will be scanned too.`n`n`n`n`n--recursive_vpk`n`n`nIf specified along with `n--recursive`n, will also recurse into VPK archives.`n`n`n`n`n--vpk_extensions`n (or `n-e`n)`n`nFile extension(s) filter, example: "vcss_c,vjs_c,vxml_c".`n`n`n`n`n--vpk_filepath`n (or `n-f`n)`n`nFile path filter(s), supports comma-separated values. Example: "panorama/,sounds/" or "scripts/items/items_game.txt".`n`n`n`n`n--vpk_cache`n`n`nUse cached VPK manifest to keep track of updates. Only changed files will be written to disk.`n`n`n`n`n--vpk_verify`n`n`nVerify checksums and signatures.`n`n`n`n`nOutput`n`n`n`n`n`n`n`n--output`n (or `n-o`n)`n`nOutput path to write to. If input is a folder (or a VPK), this should be a folder.`n`n`n`n`n--all`n (or `n-a`n)`n`nPrint the content of each resource block in the file.`n`n`n`n`n--block`n (or `n-b`n)`n`nPrint the content of a specific block, example: DATA, RERL, REDI, NTRO.`n`n`n`n`n--vpk_decompile`n (or `n-d`n)`n`nDecompile supported resource files.`n`n`n`n`n--texture_decode_flags`n`n`nDecompile textures with specified decode flags. Options: "none", "auto", "ForceLDR". Default: "auto".`n`n`n`n`n--vpk_list`n (or `n-l`n)`n`nLists all resources in given VPK. File extension and path filters apply.`n`n`n`n`n--vpk_dir`n`n`nPrint a list of files in given VPK and information about them.`n`n`n`n`nType specific export`n`n`n`n`n`n`n`n--gltf_export_format`n`n`nExports meshes/models in given glTF format. Must be either 'gltf' or 'glb'.`n`n`n`n`n--gltf_export_materials`n`n`nWhether to export materials during glTF exports.`n`n`n`n`n--gltf_export_animations`n`n`nWhether to export animations during glTF exports.`n`n`n`n`n--gltf_mesh_list`n`n`nComma-separated list of meshes to include in glTF export. By default includes all meshes in a model.`n`n`n`n`n--gltf_animation_list`n`n`nComma-separated list of animations to include in glTF export, example: "idle,dropped". Requires `n--gltf_export_animations`n. By default includes all animations.`n`n`n`n`n--gltf_textures_adapt`n`n`nWhether to perform any glTF spec adaptations on textures (e.g. split metallic map).`n`n`n`n`n--gltf_export_extras`n`n`nExport additional Mesh properties into glTF extras`n`n`n`n`n--tools_asset_info_short`n`n`nWhether to print only file paths for tools_asset_info files.`n`n`n`n`nOther`n`n`n`n`n`n`n`n--threads`n`n`nIf higher than 1, files will be processed concurrently.`n`n`n`n`n--version`n`n`nShow version information.`n`n`n`n`n--help`n`n`nShow help information.`n`n`n`n`nThere are also `n--stats`n related options (for collecting statistics and testing exports) primarily intended for VRF developers. You can pass `n--input "steam"`n to automatically scan all Steam library folders for Source 2 files. See the `n--help`n output for details.`n`nCached VPK Manifest `n​`n`n`nWhen using `n--vpk_cache`n, a `n.manifest.txt`n
 file is created alongside the VPK to track file versions. This allows 
incremental exports where only changed files are written. The cache is 
automatically invalidated if the decompiler version changes.`n`nExamples `n​`n`n`nList all files in a VPK `n​`n`n`nUse `n--vpk_dir`n to also print file metadata.`n`n`n`n`npowershell`n`n`n`n`n.`n`n/`n`nSource2Viewer-CLI.exe`n`n -`n`ni `n`n"core/pak01_dir.vpk"`n`n --`n`nvpk_list`n`n`n`n`n`nExport the entire VPK as is `n​`n`n`n`n`n`npowershell`n`n`n`n`n.`n`n/`n`nSource2Viewer-CLI.exe`n`n -`n`ni `n`n"core/pak01_dir.vpk"`n`n --`n`noutput `n`n"pak01_exported"`n`n`n`n`n`nExport only specific folders from a VPK `n​`n`n`nExport only the "panorama/layout" folder:`n`n`n`n`npowershell`n`n`n`n`n.`n`n/`n`nSource2Viewer-CLI.exe`n`n -`n`ni `n`n"core/pak01_dir.vpk"`n`n --`n`noutput `n`n"pak01_exported"`n`n --`n`nvpk_filepath `n`n"panorama/layout"`n`n`n`n`n`nDecompile and export Panorama files `n​`n`n`nDecompile and export all Panorama files to a folder named "exported":`n`n`n`n`npowershell`n`n`n`n`n.`n`n/`n`nSource2Viewer-CLI.exe`n`n -`n`ni `n`n"core/pak01_dir.vpk"`n`n -`n`ne `n`n"vjs_c,vxml_c,vcss_c"`n`n -`n`no `n`n"exported"`n`n -`n`nd`n`n`n`n`n`nPrint resource blocks `n​`n`n`nPrint resource blocks for a specific file (similar to resourceinfo.exe in Source 2).
